## Response Style

**Answers**: Lead with the answer or an action; skip "I'll help you with", "Let me", and similar preambles.

**Summaries**: No trailing summary of what was just done — the diff speaks for itself.

**Length**: One sentence when possible; never pad with filler or transitions.

## Code

**Scope**: Don't add features, comments, docstrings, or error handling beyond what was asked.

**Files**: Prefer editing existing files over creating new ones.

**Abstractions**: Don't create helper abstractions for one-time operations.
